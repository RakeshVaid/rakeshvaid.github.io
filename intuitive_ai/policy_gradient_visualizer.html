💡 What's Happening

📋 Cheatsheet

Goal: Maximize expected reward J(θ) = E[R] Policy Gradient Theorem: ∇_θ J = E[ R · ∇_θ log π(a|s) ] Loss (for gradient descent): L = −A · log π(a|s) where A = R − V(s) Gradient: ∇_θ L = −A · ∇_θ log π(a|s) = −A · ∇_θ π / π Weight Update: θ_new = θ_old − α · ∇_θ L = θ_old + α · A · ∇_θ log π(a|s) Key insight: ∇_θ log π is the direction that increases π. A > 0 → step along it → π goes UP A < 0 → step against it → π goes DOWN A ≈ 0 → barely move → π UNCHANGED Gradient magnitude ∝ |A| / π(a|s) Rare actions (small π) → large gradient → high variance problem PPO fixes this by clipping the ratio π_new/π_old to [1−ε, 1+ε]

Why does clipping exist?

The naive policy gradient loss L = −A·log π has no guardrails. A single gradient step can change the policy drastically — the model that generated the training data (πold) becomes irrelevant, and the advantages computed from that data become meaningless.

PPO's clipping creates a trust region: the new policy can only differ from the old policy by ±ε (typically 20%) per action. This means:

  • Each update is small and predictable
  • The training data stays relevant for the current update
  • No single example can cause a catastrophic policy change

After each round of updates, you throw away the data, regenerate with the new policy, and repeat. Many small trustworthy steps instead of one wild leap.

Why use the ratio πnew/πold instead of raw π?

The naive gradient ∇L = −A/π has a 1/π problem: rare actions (small π) produce enormous gradients regardless of whether the update is useful. The gradient magnitude depends on the absolute probability.

The ratio πnew/πold fixes this by measuring relative change:

  • π goes from 0.01 → 0.012 → ratio = 1.2 (20% change)
  • π goes from 0.80 → 0.96 → ratio = 1.2 (20% change)

Both are treated identically — a 20% relative shift, clipped the same way. The ratio normalizes away the absolute scale of the probability, so rare and common actions are on equal footing. No more 1/π explosion.

This is also why the ratio starts at 1.0 (πnew = πold before any update) and the clip range [0.8, 1.2] is centered on 1.0 — it directly measures "how much have we changed?"
